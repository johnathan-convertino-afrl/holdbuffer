Architecture

This core can be used with any bus. Its data width is dictated in bytes and has the ability to be forced out of hold with a timeout signal. It also has a enable to turn off all outputs of the core to mask out data. The core will buffer data in normal operation and its best case latency is one clock cycle. Its worst case is two when ready is not active (we are not ready). Usage is similar to a axi streaming interface.

  • holdbuffer decouple ready signal for pipelining.
