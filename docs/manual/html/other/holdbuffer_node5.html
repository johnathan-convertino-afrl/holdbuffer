In a Project

This can be used to pipeline any bus to bus interface. It mimicks the axi streaming one for one, but most others are similar enough to plug in without issue. The ack is added for this interfaces that require it and allows more flexable options vs just axi streaming (which includes standard AXI channels). If no timeout or enable is needed set timeout to 0 and enable to 1 and they will be ignored. If ack is needed the ACK_ENABLE parameter must be set to 1.

Adding the core to a fusesoc project, outside of adding the verilog module to your code, requires it is added as dependency. Example: 

  dep:
    depend:
      - ">=AFRL:simple:holdbuffer:1.0.0"

Module instantiation: 

  holdbuffer #(
    .BUS_WIDTH(BUS_WIDTH)
  ) inst_holdbuffer (
    .clk(clk),
    .rstn(rstn),
    .timeout(timeout),
    .enable(enable),
    .s_data(s_data_tdata),
    .s_data_last(s_data_tlast),
    .s_data_valid(s_data_tvalid),
    .s_data_ready(s_data_tready),
    .s_data_ack(s_data_ack),
    .m_data(m_data_tdata),
    .m_data_last(m_data_tlast),
    .m_data_valid(m_data_tvalid),
    .m_data_ready(m_data_tready),
    .m_data_ack(m_data_ack)
  );
